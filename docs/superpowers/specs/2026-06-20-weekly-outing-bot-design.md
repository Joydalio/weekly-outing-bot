# 주간 나들이 추천 봇 (weekly-outing-bot) — 설계 문서

작성일: 2026-06-20

## 1. 목적

만 4세 여아와 함께 갈 만한 **외출·나들이 장소**를 매주 AI가 새로 추천하고,
**매주 수요일 오전 8시(KST)**에 보호자의 휴대폰으로 **텔레그램 푸시 알림**을 보낸다.

- 대상 아동: 만 4세 여아
- 거주 지역: 서울 강동구 둔촌오륜역 인근
- 추천 방식: 매주 Claude API가 지역·계절·실시간 날씨·지역 행사를 반영해 새로 생성
- 중복 방지: 최근 약 10주간 추천된 장소는 회피

## 2. 전체 구조

접근: **Python 스크립트 + GitHub Actions 주간 크론** (무료 클라우드 실행, 맥 전원 무관).

```
[GitHub Actions 크론(수 08:00 KST)]
        │
        ▼
   generate.py  ──► Claude API (웹검색: 강동구 날씨·행사·장소)
        │                 │
        │                 ▼
        │          추천 1건 (JSON)
        ▼                 │
  history.json ◄──────────┤ (최근 10주 회피 반영)
        │                 ▼
        └────────► 텔레그램 봇 ──► 보호자 휴대폰 푸시 알림
```

## 3. 구성 요소

각 요소는 단일 책임을 가지며 독립적으로 테스트 가능하다.

| 요소 | 책임 | 의존성 |
|------|------|--------|
| `generate.py` | 전체 흐름 조율 진입점 | 아래 전부 |
| `recommender` | Claude API 호출. 입력(지역·아이 정보·날짜·최근 추천 목록) → 구조화된 추천 JSON 반환. 웹 검색으로 실시간 날씨·이번 주 행사 반영 | Anthropic SDK |
| `telegram` | 추천을 메시지로 포맷하고 봇 API로 전송 | requests, Telegram Bot API |
| `history` | `history.json` 읽기/쓰기. 최근 10주 회피 목록 제공, 신규 추천 추가 후 커밋 | json (stdlib), git |
| `config` | 지역·아이 정보·발송 시각 등 설정 보관 | — |

### 추천 JSON 스키마 (recommender 출력)

```json
{
  "place_name": "서울숲 공원",
  "intro": "사슴 먹이주기 체험과 넓은 잔디밭이 있어요.",
  "reason": "초여름 야외 활동으로 오감 자극에 딱. 만 4세는 작은 동물 관찰에 흥미가 큰 시기예요.",
  "weather_note": "수·목 맑음(26℃) 야외 추천, 금요일 비 예보",
  "map_query": "서울숲 공원",
  "travel_note": "둔촌오륜역에서 약 35분",
  "prep": "여벌 옷·간식·돗자리"
}
```

## 4. 데이터 흐름 (매주 수요일)

1. GitHub Actions 크론이 `generate.py` 실행
2. `history.json`에서 최근 10건 로드
3. Claude 호출: "강동구 둔촌오륜역 인근, 만 4세 여아, 오늘 날짜/계절, 최근 다녀온 곳 회피" + 웹 검색으로 현재 날씨·이번 주 행사 조회 → 추천 1건 생성
4. 메시지 포맷팅 + 네이버 지도 검색 링크 생성 (`https://map.naver.com/v5/search/<map_query>`)
5. 텔레그램 전송
6. `history.json`에 `{date, place_name}` 추가 후 커밋·푸시

## 5. 메시지 형식

이모지는 핵심 위치(장소·지도)에만, **최대 3개**까지만 사용한다. 나머지 구분은 굵은 글씨와 줄바꿈으로 처리한다.

```
**이번 주 나들이 추천 · 6/24**

📍 **서울숲 공원**
사슴 먹이주기 체험과 넓은 잔디밭이 있어요.

왜 좋은가 — 초여름 야외 활동으로 오감 자극에 딱. 만 4세는 작은 동물 관찰에 흥미가 큰 시기예요.

날씨 — 수·목 맑음(26℃) 야외 추천, 금요일 비 예보

🗺️ [지도 보기](네이버지도 링크) · 둔촌오륜역에서 약 35분

준비물 — 여벌 옷·간식·돗자리
```

사용 이모지: `📍`(장소), `🗺️`(지도) 2개. 필요 시 날씨에 1개 추가 가능(최대 3개).

## 6. 에러 처리

- **Claude 생성 실패**: 1회 재시도. 그래도 실패하면 "이번 주 추천 생성에 실패했어요" 안내 메시지를 텔레그램으로 전송해 조용한 누락 방지
- **텔레그램 전송 실패**: GitHub Actions가 실패(빨간 X)로 표시 → Actions 탭에서 확인. (선택) 1회 재시도
- **history.json**: 단일 작성자라 충돌 거의 없음

## 7. 테스트 전략

- **`--dry-run` 모드**: 실제 전송·기록 없이 콘솔에 메시지만 출력 (개발 확인용)
- **수동 실행**(workflow_dispatch): 수요일까지 안 기다리고 GitHub에서 즉시 테스트 발송
- **단위 테스트** (Claude·텔레그램은 목으로 대체):
  - 메시지 포맷팅이 명세대로 출력되는가 (이모지 ≤ 3)
  - 네이버 지도 링크가 올바르게 생성되는가
  - history 읽기/쓰기 및 최근 10주 회피 목록 산출이 정확한가

## 8. 기술 스택 & 사전 준비물

- Python 3.11, `anthropic` SDK, `requests`, 표준 `json`
- GitHub Actions 워크플로 (cron: `0 23 * * 2` UTC = 수요일 08:00 KST)

### 보호자가 1회 준비할 것 (구현 후 안내서 제공)

1. 텔레그램 봇 생성(@BotFather) → 봇 토큰 확보
2. 본인 chat_id 확보 (봇에게 메시지 후 getUpdates 조회)
3. Anthropic API 키 발급
4. GitHub 저장소 생성 후 시크릿 3개 등록: `ANTHROPIC_API_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`

## 9. 범위 밖 (YAGNI — 향후 확장 후보)

- 봇 양방향 응답("여기 갔다왔어/별로였어" 피드백 반영)
- 발달·놀이/집콕 만들기 등 외출 외 카테고리
- 다중 아동·다중 지역 지원
- 이미지 첨부
